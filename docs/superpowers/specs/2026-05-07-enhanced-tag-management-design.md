# 事件标签管理优化设计文档

## 1. 背景与目标

当前系统中标签的管理不够便捷：详情页不支持直接删除，且无法在关联标签时直接创建库中不存在的新标签。

**目标：**

- 支持在详情页侧边栏一键移除标签。
- 移除操作提供“撤销”通知。
- 支持在关联标签时输入新文本并按回车直接创建新标签。
- 新标签自动分配随机颜色。

## 2. 方案设计

### 2.1 详情页直接删除 (EventDetailView.vue)

- **UI 修改**：侧边栏标签展示项增加 `X` 按钮。
- **交互逻辑**：
  1. 点击 `X` 后，立即更新本地状态移除该标签。
  2. 调用后端 `updateEventTags` 接口同步更改。
  3. 弹出 `naive-ui` 的 `message` 或专门的 `undo` 通知，允许用户撤销删除。
- **撤销逻辑**：利用项目中已有的 `useUndo` 逻辑或手动实现，将移除的 ID 重新加回。

### 2.2 标签选择器优化 (Tag Selector Modal & Form Modal)

- **组件更新**：将 `Tag Select Modal` (详情页) 和 `EventFormModal` (创建/编辑页) 中的标签选择器改为 `n-select`。
- **配置项**：
  - `multiple`: 支持多选。
  - `filterable`: 支持搜索。
  - `tag`: 核心功能，支持将输入内容转换为标签。
- **保存逻辑**：
  1. 遍历选中的标签。
  2. 如果标签是新输入的文本（非 ID），先调用 `createTag` 接口创建标签（分配随机颜色）。
  3. 获取所有标签的 ID 列表。
  4. 调用 `updateEventTags` 进行关联。

### 2.3 颜色随机化策略

预设一组符合系统视觉风格的柔和色值（Archival/Muted 色调），在创建新标签时随机抽取一个。

```typescript
const PRESET_COLORS = ['#71717a', '#14b8a6', '#eab308', '#f43f5e', '#22c55e', '#3b82f6'];
```

## 3. 涉及文件

- `packages/web/src/views/EventDetailView.vue`: 侧边栏交互与标签模态框更新。
- `packages/web/src/components/EventFormModal.vue`: 创建/编辑弹窗中的标签选择器更新。
- `packages/web/src/api/tagApi.ts`: 确保 API 支持。

## 4. 测试方案

- **详情页删除**：点击标签 X 按钮，确认标签消失且数据库同步；点击撤销，确认标签恢复。
- **创建新标签**：在 `n-select` 中输入一个从未出现的词（如“新记忆”），保存后确认该标签已创建且成功关联。
- **随机颜色**：多次创建不同新标签，确认它们获得了不同的预设颜色。
