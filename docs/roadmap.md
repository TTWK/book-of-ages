# 开发计划 (Roadmap)

基于系统作为“AI 友好的个人数据底座”这一核心定位，开发计划将分为以下四个阶段（Phases）进行迭代推进。

---

## Phase 1: 核心中台与 API (后端优先)

**目标：** 建立精简后的数据库模型，实现核心的事件/草稿 CRUD、API Key 鉴权以及 URL 基础解析工具，为前端和外部 Agent 提供坚固的数据基础。

### 后端 (Server)
- [ ] 项目脚手架与基础框架搭建（Fastify + TypeScript + 目录结构设计）
- [ ] 数据库连接与初始化脚本（基于 `better-sqlite3` 创建简化的表结构）
- [ ] 基础 API 开发：
  - [ ] 事件 CRUD API (包含 `status` 草稿支持)
  - [ ] 标签体系 CRUD API
- [ ] 系统与安全审计：
  - [ ] `api_keys` 管理接口
  - [ ] API 请求鉴权中间件 (验证 `X-API-Key`)
  - [ ] 简易 `operation_logs` 拦截写入逻辑
- [ ] **增强工具**：
  - [ ] URL 解析服务 (`/api/tools/parse-url`): 利用 Readability 提取标题和 Markdown 正文

### 前端 (Web)
- [ ] 前端项目初始化（Vue 3 + Vite + Naive UI）
- [ ] 基础网络请求封装（Axios + 统一错误处理拦截）
- [ ] 路由搭建与空页面占位

---

## Phase 2: 收件箱与审批流 (前端优先)

**目标：** 构建前端 UI 核心体验，打通“外部 Agent 推送 -> 人工浏览 -> 确认收录”的完整闭环链路。

### 前端 (Web)
- [ ] **收件箱 (Inbox) 看板开发**：
  - [ ] 拉取并展示 `status='draft'` 的待处理事件列表
  - [ ] 提供快速预览事件梗概的能力
  - [ ] 提供一键“收录 (Confirm)” 与 一键“忽略 (Archive/Delete)”操作
- [ ] **正式事件库视图**：
  - [ ] 展示已确认 (`confirmed`) 的时事列表
  - [ ] 列表分页、基于基础状态的筛选
- [ ] **系统设置视图**：
  - [ ] 提供管理、生成 API Key 的 UI 界面

### 后端 (Server)
- [ ] 配合前端联调，修复可能的 API 返回格式和分页相关 Bug
- [ ] 编写测试脚本，模拟外部 Agent 推送数据，验证 Inbox 流程

---

## Phase 3: 标签聚类与全文检索

**目标：** 实现完善的标签分类展示、事件详情查阅与基于 SQLite FTS5 的全局搜索功能，提升系统内数据被定位的效率。

### 后端 (Server)
- [ ] 部署 SQLite FTS5 虚拟表及数据同步触发器
- [ ] 编写 `/api/search` 全局搜索接口
- [ ] 标签聚合查询逻辑优化（如按标签查事件列表）

### 前端 (Web)
- [ ] **事件详情页开发**：
  - [ ] 详情展示与 Markdown 渲染
  - [ ] 支持编辑事件正文、修改关联标签
- [ ] **标签管理侧边栏/视图**：
  - [ ] 树形标签展示与快捷筛选
- [ ] **全局搜索交互**：
  - [ ] 顶部全局搜索框
  - [ ] 搜索结果高亮展示与跳转

---

## Phase 4: 深度资料管理与时间线

**目标：** 提供深度的事件脉络追踪能力，允许挂载多媒体参考材料（Materials）与时间线节点（Timeline Nodes）。

### 后端 (Server)
- [ ] 时间线节点 CRUD API (`/api/events/:id/timeline`)
- [ ] **文件系统与材料 API**：
  - [ ] 实现安全的文件上传机制与本地目录归档 (`/api/materials/upload`)
  - [ ] 材料查询、删除与预览服务 (`/api/materials`)
  - [ ] URL 解析快照保存功能集成至材料模块

### 前端 (Web)
- [ ] **事件详情页增强**：
  - [ ] 内嵌时间线组件，支持图形化查看和追加发展脉络
  - [ ] 内嵌参考材料画廊/附件列表，支持上传新的网页快照、图片或 PDF 文件
- [ ] **剪藏增强体验**：
  - [ ] 在前端界面提供手动输入 URL，一键调取后端解析接口，并将结果保存为 Markdown 或快照的功能

---

- [ ] 完善 Docker 容器化配置（`Dockerfile` 及 `docker-compose.yml`）
- [ ] 撰写面向普通用户的 NAS / 本地服务器一键部署文档
- [ ] 提供数据库与附件目录自动备份的配置建议或脚本
