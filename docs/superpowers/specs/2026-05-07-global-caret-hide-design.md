# 针对性隐藏非输入文本光标 (Caret) 设计文档

## 1. 背景与目标

在项目中，部分仅用于展示的文本（如标题、描述、Logo 等）在被点击或聚焦时会出现文本输入光标（Caret），这会给用户带来“该文本可以编辑”的错觉。

**目标：**

- 针对性隐藏展示类文本（如标题、导航、Logo、说明文字等）的光标。
- 避免全局修改带来的潜在风险，确保输入框功能完全不受影响。

## 2. 方案设计

### 2.1 CSS 策略

采用“常用标签自动覆盖 + 特定元素手动应用”的结合方案。

1. **常用标签覆盖：** 在全局样式中针对 `h1`, `h2`, `h3` 以及项目中定义的 `.text-h1`, `.text-h2`, `.text-h3` 类应用 `caret-color: transparent`。标题通常不具备输入功能。
2. **工具类定义：** 定义 `.no-caret` 工具类，用于灵活应用于其他非标题类的展示文本。

### 2.2 修改内容

#### 全局 CSS (`packages/web/src/style.css`)

```css
/* 针对标题隐藏光标 */
h1,
h2,
h3,
.text-h1,
.text-h2,
.text-h3 {
  caret-color: transparent;
}

/* 通用隐藏光标工具类 */
.no-caret {
  caret-color: transparent;
}
```

#### 组件应用

在以下组件的关键元素上添加 `.no-caret` 类：

- **`MainLayout.vue`**: Logo 容器、导航按钮中的文本。
- **`EmptyState.vue`**: 标题和描述容器。
- **`EventsView.vue`**: 副标题描述。
- **`EventCard.vue`**: 摘要描述、日期标签。

## 3. 影响评估

- **安全性:** 相比全局方案，该方案不会触及任何 input 或 textarea 元素，风险极低。
- **扩展性:** 后续新增的展示类文本可以直接通过添加 `.no-caret` 类来解决。

## 4. 测试方案

- **手动验证:**
  - 点击页面标题（H1/H2）、卡片文本，确认不出现光标。
  - 点击 Logo 和导航，确认不出现光标。
  - 点击“记录新事件”弹窗中的输入框，确认光标正常。
