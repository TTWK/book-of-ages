# 2026-05-06 优化 Header 导航栏方案

## 背景 (Context)
当前应用在桌面端由于 Header 导航栏菜单项过多，导致在小分辨率屏幕或缩放窗口时出现横向滚动条。这不仅影响视觉统一性（Design Token 应用不彻底的表现），也降低了操作效率。

## 目标 (Goals)
1. 消除桌面端 Header 的横向滚动条。
2. 保持导航逻辑的一致性。
3. 遵循现有的设计系统（Book of Ages）。

## 设计方案 (Design)
### 1. 冗余项清理
经分析，导航项中的“搜索”项与 Header 右侧的搜索输入框功能完全重复。在移动端，顶部 Header 也已经有独立的搜索入口。
**决定**：从 `navItems` 配置中移除“搜索”项。

### 2. 布局优化
移除一个导航项后，剩余项将获得更多空间，足以在标准 1024px+ 分辨率下无滚动显示。

## 涉及文件 (Files Affected)
- `packages/web/src/components/MainLayout.vue`

## 验证标准 (Verification)
1. 桌面端（窗口宽度 > 768px）Header 不再出现横向滚动条。
2. 点击右侧搜索框/回车仍可正常进入搜索页。
3. 移动端底部 Tab 栏移除搜索项后，顶部搜索按钮依然有效。
