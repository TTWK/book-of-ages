# 2026-05-06 优化 Header 布局与菜单结构方案

## 背景 (Context)

虽然之前精简了一个导航项，但在小分辨率下，搜索框聚焦时的宽度拉伸 (`focus:w-56`) 依然会挤压左侧剩余的 5 个导航项，导致横向滚动条出现。用户希望进一步精简，避免使用下拉菜单，并倾向于将非核心项降级或整合进二级页面。

## 目标 (Goals)

1. 将 Header 导航项减少到最核心的 3 项，彻底消除空间冲突。
2. 不引入新的下拉菜单。
3. 保持功能入口可达。

## 设计方案 (Design)

### 1. 导航项降级与迁移

- **移除** `navItems` 中的“标签”和“时间分析”。
- **“时间分析” (Analytics)**: 迁移至 Header 右侧操作区，仅以图标形式展示。
- **“标签” (Tags)**: 迁移至“事件库”页面内部。在页面头部增加“标签管理”入口。

### 2. 最终 Header 结构 (桌面端)

- **左侧**: Logo
- **中间**: 事件库、收件箱、时间线 (3项)
- **右侧**: 搜索框 | 深色模式 | 时间分析(新) | 操作审计 | 设置

## 涉及文件 (Files Affected)

- `packages/web/src/components/MainLayout.vue`: 修改 `navItems` 数组，在 Actions 区添加 Analytics 按钮。
- `packages/web/src/views/EventsView.vue`: 在 Header 增加“标签管理”跳转按钮。

## 验证标准 (Verification)

1. 搜索框聚焦时，Header 导航区不再出现滚动条。
2. 点击右侧“时间分析”图标可正常跳转。
3. 进入“事件库”页面可看到“管理标签”按钮并成功跳转。
