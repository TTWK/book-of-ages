# 开发路线图

## ✅ 已完成

### Phase 1-4：收集 → 整理

| 功能             | 说明                                         |
| ---------------- | -------------------------------------------- |
| **事件 CRUD**    | 创建、查看、编辑、软删除事件                 |
| **收件箱审批**   | Agent 推送草稿 → 收件箱 → 一键收录/忽略      |
| **URL 一键剪藏** | 输入 URL 自动提取标题和正文                  |
| **时间线管理**   | 为事件添加/编辑/删除时间线节点，梳理事件脉络 |
| **参考材料**     | 上传图片、文档、网页快照等附件               |
| **标签体系**     | 树形标签，创建事件时自动创建新标签           |
| **全文搜索**     | SQLite FTS5 全文检索，结果高亮               |
| **API Key 管理** | 生成、查看、吊销 Agent 密钥                  |
| **操作审计**     | 记录所有 API 和 UI 操作日志                  |
| **Docker 部署**  | 前后端容器化，适合 NAS 部署                  |
| **CI/CD**        | GitHub Actions 自动检查代码质量              |

## ⏳ 待规划

### Phase 5：导出与分析

**核心目标**：从"收集整理"走向"输出洞察"。

| 功能             | 描述                                                                 | 优先级 |
| ---------------- | -------------------------------------------------------------------- | ------ |
| **事件叙事导出** | 将事件及其时间线、材料、标签整合为一份完整的叙事文档（Markdown/PDF） | P0     |
| **时间聚合分析** | 按周/月/年聚合事件，展示趋势和密度热力图                             | P0     |
| **标签聚合分析** | 按标签聚类，展示某类事件的完整时间线                                 | P1     |
| **批量导出**     | 选中多个事件，打包导出为压缩包                                       | P1     |
| **定期报告**     | 自动生成周报/月报，汇总近期事件                                      | P2     |

### Phase 6：远期

| 功能               | 描述                                      |
| ------------------ | ----------------------------------------- |
| **多用户协作**     | 账号体系 + 权限分级，支持多人共同维护     |
| **数据导入/迁移**  | 从其他系统（Notion/Obsidian/CSV）导入数据 |
| **数据清理策略**   | 冷归档 + 物理删除，管理存储压力           |
| **移动端适配优化** | 更好的移动端体验                          |
| **插件系统**       | 扩展 URL 解析器、导出格式等               |
