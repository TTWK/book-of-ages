# 岁月史书 (Book of Ages) - 核心设计文档

**文档日期**: 2026-04-02
**项目目标**: 构建一个作为“纯净数据底座”的个人热点时事记录工具，提供极其高效的事件浏览与手动录入体验，同时通过提供对 AI 友好的 API，配合外部 Agent（如定时脚本）实现自动化信息采集、提炼和补充。

## 1. 系统架构与核心流程

本系统（Book of Ages）是一个高内聚的 Headless CMS（无头内容管理系统）与个人 UI 展示端的结合体。

### 核心流转机制 (Events & Inbox Flow)
1. **正式事件库（核心）**：用户在 Web 首页最显眼的位置浏览已确认的正式事件库。用户可以**手动创建**事件，手动创建的事件默认状态为 `confirmed`（已收录），这是系统最核心和高频的交互场景。
2. **自动搜寻（外部 Agent）**：外部爬虫/时事 Agent 定时运行，搜寻并总结当天的热点事件。
3. **收件箱推送（API）**：Agent 通过带有鉴权（API Key）的 RESTful API，将抓取和总结的新事件以 `status='draft'` 的状态推送到本系统的“收件箱 (Inbox)”。
4. **人工审批与边界控制**：
   - 用户在收件箱中浏览 Agent 推送的事件，点击“确认收录 (Confirm)”，事件变更为 `confirmed` 并进入正式库。
   - **严格的修改边界**：一旦事件被标记为 `confirmed`，外部 Agent 将**彻底失去**修改该事件“主标题 (title) 和 主内容 (content)”的权限。此后，事件的主体信息仅允许用户在 UI 端手动编辑。
5. **深度资料补全（Timeline & Materials）**：在事件确认收录后，外部 Agent 仍可通过 API 继续向该事件追加细化的时间线节点（Timeline Nodes）或参考材料（Materials）。用户在前端同样可以自由地对时间线和参考材料进行手动增删改。

## 2. 核心功能设计

### 2.1 极简且核心的录入体验 (UI First)
- **手动录入优先**：系统的第一屏和核心心智是“我的事件库”。“手动新建事件”的优先级高于“处理收件箱”。
- **所见即所得的 URL 解析 (`/api/tools/parse-url`)**：
  - 用户或 Agent 均可调用此服务。在 UI 层面，用户输入 URL 后，系统提取网页标题和正文并**填充至当前的表单中**，用户在表单里确认并可做任意调整，最后再点击保存。
- **无感知的标签创建**：用户在编辑或新建事件时，若输入了系统中不存在的标签，系统会在保存事件时自动创建该新标签并建立关联，确保思路不被打断。

### 2.2 AI 友好的 API 接口与鉴权底座
- **严格鉴权**：系统保留 `api_keys` 表，为不同的外部 Agent 分配特定的 Token。
- **高容错与规范响应**：所有的 API 必须提供极度规范的 JSON 输入输出格式，并带有明确的错误代码。
- **基础防灾审计**：保留简化的 `operation_logs` 表，仅记录审计级别的动作（“哪个 API Key 在什么时间对什么实体进行了何种 CRUD 操作”），用于追踪异常行为。

### 2.3 极简数据模型 (YAGNI 原则调整)
为了保证系统能快速交付，对原有的复杂数据模型进行以下精简：
- **移除复杂的版本控制**：删除 `event_versions` 表。
- **简化事件关联**：删除 `event_relations` 表。前期主要依靠“标签体系 (Tags)”来进行事件聚类。

## 3. 技术栈保持不变

| 层级 | 技术选型 | 说明 |
|------|------|------|
| **前端 UI** | Vue 3 + Vite + Naive UI | 提供直观的事件列表、收件箱看板、标签管理与详情展示 |
| **后端 API** | Fastify (TypeScript) | 提供极速、规范的 RESTful API |
| **数据存储** | SQLite + 文件系统 | 基于 `better-sqlite3`，本地单文件存储，易于备份与 NAS 部署 |
| **全文检索** | SQLite FTS5 | 用于对事件标题、Markdown 正文与摘要进行极速搜索 |

## 4. 后续开发计划建议（Phase 调整）

- **Phase 1: 核心中台与 API (后端优先)**：建立精简后的数据库模型，实现核心的事件/草稿 CRUD、API Key 鉴权以及 URL 基础解析工具。
- **Phase 2: 核心事件库与收件箱 (前端优先)**：构建“事件列表视图”、“手动创建/编辑事件”、智能标签录入，其次搭建“收件箱 (Inbox) 待处理看板”。
- **Phase 3: 标签聚类与全文检索**：实现完善的标签分类展示、事件详情查阅与基于 SQLite FTS5 的全局搜索功能。
- **Phase 4: 深度资料管理与时间线**：实现多媒体参考材料（Materials）与时间线节点（Timeline Nodes）的前后端全功能（包含 Agent 自动化追加与用户手动自由编辑）。
- **Phase 5 (Future): 用户体系与多端协作**：考虑加入多用户系统、鉴权登录以支持公网安全访问。
